## 二、字体 / 样式硬规则（v2.4 新增）

### S-1 字体堆栈（中日双语必用）

```css
:root {
  --font-serif: "Noto Serif SC", "Noto Serif JP", "Songti SC",
                "STSong", "Hiragino Mincho ProN", "Yu Mincho", serif;

  /* 装饰字体仅用于标题/印章·不要全局用 */
  --font-decorative: "Shippori Mincho", "Klee One", "Songti SC", serif;
}
```

**禁忌**：用纯日文字体（如 Shippori Mincho）做主体·简体字 glyph 缺失会 fallback 到楷书。

### S-2 timeline 上下两层（不是左右）

```css
.tl-item {

  padding: 4px 12px 8px;
}
.tl-time {
  display: inline-block;
  font-family: ui-monospace, "SF Mono", monospace;
  font-size: 12px;
  margin-bottom: 6px;
  padding: 2px 10px 2px 6px;
  border-left: 3px solid var(--hl);
  background: linear-gradient(90deg, rgba(176,58,46,.08), transparent);
}
.tl-body {
  margin-left: 0 !important;
  width: 100%;
}
```

### S-3 day-nav grid 等分（不要横向滚动）

```css
.day-nav {
  display: grid !important;
  grid-template-columns: repeat(N, 1fr);  /* N = 天数 */
  gap: 4px;
  overflow: hidden !important;
}
```

### S-4 链接按钮 outline 风

```css
.link-primary, .link-gaode, .link-maps, .link-fallback, .link-ig, .link-video {

  color: var(--hl);

  border-radius: 2px;
}
.link-primary { background: var(--hl); color: var(--bg); }  /* 主按钮才实心 */
```

### S-5 删除预算追踪 default

```css
#quick-cards { display: none; }  /* 默认隐藏 */

```

## 三、工程规则

### W-1 交付节奏（金标准）

| 数据量 | 交付方式 |
|---|---|
| 1-5 张 cards | 一次做完 |
| **6-15 张** | **先 1 张样板 → 用户验收 → 1 天 1 天交付** |
| **16+ 张** | **T1 骨架 → T2 核心数据 → T3 cards 1 天 1 天** |

**铁律**：用户主动建议"按 1 天 / 3 天拆"是金标准·不要赶。

### W-2 写数据前必读渲染函数

进 Stage 3 第一件事·读完：
- `renderCard()`
- `renderTimelineItem()`
- `renderDay()`
- `renderDayMapThumb()`
- `collectDayCoords()`
- `renderStickyCtx()`
- `renderQuickCards()`

把它们消费的所有字段列成 schema·**作为 cards 数据的写作模板**。

### W-3 内联 trip-data 优先（双击即用）

```html
<!-- 在主 script 之前 -->
<script id="trip-data" type="application/json">
{...完整 trip.json...}
</script>

<script>
async function init() {
  const inlineEl = document.getElementById('trip-data');
  if (inlineEl && inlineEl.textContent.trim().length > 10) {
    TRIP = JSON.parse(inlineEl.textContent);
  } else {
    TRIP = await fetch('trip.json').then(r => r.json());  // 兜底
  }
  // ...
}
</script>
```

### W-4 inline JSON 注入用 str.replace（不用 regex）

```python
# ❌ 错（regex lazy 匹配会从注释里的字符串扫起）
content = re.sub(r'<script id="trip-data"...>.*?</script>', new_block, content, flags=re.DOTALL)

# ✅ 对（字符串替换）
old_block = content[content.index('<script id="trip-data"'):content.index('</script>', ...)+9]
content = content.replace(old_block, new_block)

# ✅ 也对（regex 用 lambda·避免反向引用问题）
content = pattern.sub(lambda m: new_block, content)
```

## 四、事实核查清单

### F-1 餐厅 4 必查

每个餐厅写卡前用 web_search 验证：

1. **定休日**（特别看是否撞行程日）
2. **营业时间**（开门 / 关门 / L.O.）
3. **是否接受预约**
4. **末班 LO 时间**（影响晚饭安排）

### F-2 用户特定地名必查

用户说"X 站去 Y 寺"·"小渕町坐车去 X"等具体动线时：
- 全部 web_search 验证
- 不要靠常识猜
- 巴士 / 电车具体班次也要查

### F-3 关键店铺验证清单（标准）

```
店名 + 地址 + 营业时间 + 定休日 + 价位 + 历史
```

每张 Tier A 卡至少 1 次 web_search·关键店（已订位的）至少 2 次（一次查时间·一次查菜单/历史）。

## 五、Stage 3 执行流程（推荐）

```
1. 读 trip.template.html 全部 render 函数（30 分钟）
2. 列出 cards schema（按本文档第一章）
3. T1 骨架（CSS + HTML + 空 TRIP_DATA）
4. T2 meta + days timeline 主轴
5. T3 cards · 第一天作为样板（5-12 张完整字段）
   ↓ 用户验收
6. T3 cards · D2 D3 D4 D5 1 天 1 天交付
   ↓ 每天验收
7. T4 todo-before-trip.md
8. 关 HTTP server / 复盘
```

## 七、链接策略硬规则（v2.4.1 升级）

### 7.1 禁用清单（绝对不写进 cards.links）

| 类型 | 站点 | 原因 |
|---|---|---|
| 国内点评 | 大众点评（dianping.com）| 强制登录·点开跳登录页 |
| 国内点评 | 美团（meituan.com）| 强制登录 + 部分商户跳 App |
| 国内社交 | 小红书（xiaohongshu.com）| 强制登录看完整笔记 |
| 国内社交 | 微博（weibo.com）| 强制登录看完整内容 |
| 国内站点 | 抖音网页版（douyin.com）| 强制登录看视频 |
| 国际社交 | 微信公众号链接 | 微信外打不开 |

### 7.2 推荐链接库（按优先级）

**国内目的地：**
1. 高德地图 `https://www.amap.com/search?query={name}&city={city}`（万能兜底）
2. 官网（如有）
3. 携程 ctrip.com
4. 黑珍珠（meituan.com 子页面但不强登）— 实际仍要登录·**待验证**

**国际目的地：**
1. Google Maps `https://www.google.com/maps/search/?api=1&query={name}`（万能兜底）
2. 官网
3. Tabelog（日本餐厅·tabelog.com）— 不需要登录
4. TripAdvisor（tripadvisor.com）— 不需要登录
5. Instagram（公开账号）— 不需登录可看
6. YouTube — 不需登录可看

### 7.3 链接验证流程

写入每张 cards.links 前·必做：

```bash
# 在浏览器隐身/无痕模式打开链接·确认能看到内容·不跳登录页
# 如果跳登录页·立即从 links 删除·改成 Maps 兜底
```

## 八、Stage 3 最终对账步骤（v2.4.1 新增）⭐

**进 Stage 3 渲染完毕后·必须执行下面对账·全部通过才告诉用户「完成」：**

### 8.1 渲染功能自检（在浏览器实测）

打开 trip-final.html·逐项验证：

- [ ] **顶栏 hero**：目的地+天数+人数+出发地都正确显示
- [ ] **sticky-ctx**：电话/语言/支付提示有内容（不是占位）
- [ ] **4 个 modal**：点开都有内容·不空白
  - [ ] 紧急（emergency 数组·≥ 5 条）
  - [ ] 须知（reminders 数组·≥ 6 条）
  - [ ] 准备（preparation 数组·≥ 15 条·可勾选）
  - [ ] 票据（tickets_seed 数组·≥ 5 条预填）
- [ ] **Day 导航**：N 天等分胶囊·不滚动·点击跳转工作
- [ ] **Day 主区**：N 天 day-section 全部渲染
  - [ ] 天气栏有真实数据（不是占位文字）
  - [ ] reminders 黄色提示条显示
  - [ ] timeline 全部 slot 渲染
  - [ ] critical 节点红色高亮 + 关键原因
  - [ ] outdoor 卡片有 🌤 室外标签
  - [ ] 餐饮汇总 / 备用方案 / 当日备注 三栏存在
- [ ] **每张卡片**：
  - [ ] name + emoji + city_label
  - [ ] highlight 一行短亮点
  - [ ] signature_dishes（餐厅）或 key_info
  - [ ] 链接按钮 outline 风
  - [ ] 「展开介绍」按钮·点开有 ≥ 2 段
- [ ] **迷你地图**：每天 day section 顶部小地图能渲染（说明 coords 没漏）
- [ ] **alt_candidates**：每天底部「附近可替换」区域显示备选
- [ ] **Edit Mode**：顶部铅笔图标点击进入编辑模式·能改字段
- [ ] **字体**：标题+正文粗细统一·没有楷书 fallback
- [ ] **双击 file://**：直接打开能看到完整页面（不依赖 HTTP server）

### 8.2 链接验证

- [ ] 随机抽 5 张 cards 的 links·**手机无痕模式打开**·全部不跳登录页
- [ ] 不应该出现 dianping / meituan / xiaohongshu / weibo

### 8.3 数据完整性

- [ ] 全部 cards 有 intro_paragraphs（≥ 1 段）
- [ ] 全部餐厅 cards 有 signature_dishes（≥ 3 道）
- [ ] 全部 cards 有 key_info（餐厅）或 hours（景点）
- [ ] 关键 cards 有 coords（让地图渲染）
- [ ] weather 5 天有真实数据（不是「待 X 后查」占位）

### 8.4 流程

```
渲染完毕 → 自检 8.1-8.3 → 全部通过 → 告诉用户完成
                          ↓ 任意未通过
                       写补丁脚本修复 → 复测
```

**铁律：自检不通过·不要说「完成了」。**

> 京都 case 教训：4 个 modal 全空·我直到用户提醒才发现。如果 Stage 3 后做 5 分钟自检·应该能抓到。
