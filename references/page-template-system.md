# 行程页面模板系统 · 设计规格 v1（draft）

> 核心原则：**模板由「当天行程的形状」决定，不由内容主题决定。一天可拆成多页。**
> 两层设计：① 原子模板（页面单元）② 一天 → 几页的组合逻辑。

---

## 一、图片 & 地图形状铁律（所有模板遵守）

- **图片槽**：只用 16:9（图在上、文字在下）或 4:3（图在左、文字在右，图 ≥38% 宽）。**禁止竖窄条 / 横窄缝。**
- **地图槽**：用于点位标注的地图必须接近 **方形或横向（1:1 / 4:3 / 3:2）**，能铺得下 marker。**禁止窄条地图。**
- 徒步线路图：近 1:1，可纵向带高程剖面。

## 三、一天 → 几页（组合逻辑，按优先级判断）

| 形状 | 拆页 |
|---|---|
| 单城、点位密（≥4，超容量见 §四） | T1 ×N |
| 整天同质（≥80% 同类，见 §五-Q3） | T4 ×N |
| 1–3 个松散行程 | T5 / T6 / T7（按数量） |
| 跨城 + 落地 ≤2 点 | T2 单页（落地点并入页底） |
| 跨城 + 落地 ≥3 点 | T2 + T1/宽松 |
| 半天徒步 + 其他 | T3 + T1/宽松 |
| 跨城 + 落地即徒步 | T2 + T3 |
| 多段跨城为主（纯赶路） | T2 单页 |
| 整天无安排（休整） | T8 单页 |

## 四、拆页机制（解决"1 页满 1 页空"）—— Q1

- 每页软上限 **maxPerPage = 5**。
- 页数 `pages = ceil(N / 5)`。
- **均衡分配**，不是塞满前页：N 个点平均分到 pages 页，差值 ≤1。
  - 6 → 3+3｜7 → 4+3｜8 → 4+4｜9 → 5+4｜11 → 4+4+3。
  - **绝不出现 5+1。**
- 切点优先落在**自然时间段边界**（上午 / 下午 / 晚上）；先算均衡份数，再把切点微调到最近的时间空档。
- 餐饮随**所覆盖时间段**进对应页（Q2 锁定）：早午餐进上半页、晚餐进下半页。
- 片区切分**不在本环节处理**——Stage 2 行程规划已引导用户按片区组织。

## 五、待写进 Schema 的定义缺口

### Q3 · 同质类型怎么判定（需补进 card.category）
单一主题日(T4) 有**两个触发条件，命中任一即可**：
- **(a) 类型同质**：排除交通后 **≥80% 属同一 category**（逛街/扫街吃/逛校园）。
- **(b) 空间高度集中**：当天活动范围极小、点位扎堆在**一个街区 / 一个古镇 / 一个园区**内——即使类型不完全相同，也按 T4 处理（步行可达、无需地图导航跳转）。需 Stage 2 标记 `compact_area`（bool + 范围名）。

每个 card 打一个 `category`：
建议分类（≈7 类，去重）：

| category | 含义 | 例 |
|---|---|---|
| sightseeing | 观光景点 | 古迹、地标、观景台 |
| culture | 文化场馆 | 博物馆、美术馆、剧院 |
| food | 餐饮美食 | 餐厅、夜市、扫街 |
| shopping | 购物 | 商场、市集、买手店 |
| outdoor | 户外自然 | 公园、海岸、山林 |
| experience | 体验活动 | 出海、手作课、演出、乐园 |
| venue | 特定场所 | 校园、园区、特定打卡主题 |

> 交通(transit) 不计入分母。1–3 个仍优先按宽松(§三)处理，同质判定只用于 ≥4 个点的日子。

### Q5 · 跨城落地点溢出（用哪个）
- 落地后 **≤2 点** → 直接并进 T2 页底（不另起页）。
- 落地后 **≥3 点** → 另起一页：落地点 ≥4 且不同质 → T1；1–3 个 → 宽松；≥4 同质 → T4。

### Q6 · 徒步日的吃（Stage 2 ↔ 渲染 两边同步设计）
- 徒步日常常**回程才吃饭**，徒步中途不打卡吃东西。
- Stage 2 需新增字段：徒步线路上**是否有补给/进食点**。
  - 有 → 作「补给点」放进 T3 徒步页内。
  - 无 → 当天正餐安排放到配套的下一页（T1/宽松）。

### Q7 · 休整日 = 新增 T8 空状态模板
- 纯赶路 → T2 单页（不新增）。
- 整天无安排（休整）→ T8：现有模板都依赖配图，休整日没有打卡点，需**无照片的空状态版式**（插画/纹理 + 「今日自由」+ 可选餐饮/周边闲逛建议）。

## 八、锁定规则（驱动自动选模板）

### A · type → category 映射表（同质判定用 category）
| 现有 type | → category |
|---|---|
| sight | sightseeing |
| sight_cultural | culture |
| sight_nature, experience_outdoor | outdoor |
| meal_breakfast/lunch/dinner/snack, experience_food | food |
| shop | shopping |
| experience | experience |
| venue（新增） | venue |
| free | （按实际内容归类，默认 experience） |
| transit, hotel | 不计入分母（除非按 §九 升格为点位） |

**新增 type `venue` 的边界（不与现有类型重叠）：**
- venue = **场所/园区型目的地**：大学校园、产业园区、大学城、围合式主题园区——你去是为了「这个场所本身」。
- 不重叠：单体景观（塔/桥/观景台）→ sight｜博物馆/美术馆 → sight_cultural｜一次性活动（出海/课程/演出）→ experience｜游乐场 → experience。
- 判别句：「它是一个可漫步的围合场所，而非单一景点或单一活动」→ venue。

### B · 空间集中 = 坐标自动算，分多档（不问用户）
取当天所有「点位」(§九) 的 `card.coords`，算两两**最大直线距离**，分档（直线距离为代理值，规划阶段无路由 API）：

| 档位 | 直线距离代理 | 约等于 | 对模板的影响 |
|---|---|---|---|
| 步行圈 | ≤ 1.5km | 步行 ~20min | **触发 T4 compact**（即使不同质） |
| 打车圈 | ≤ 5km | 打车 ~20min | 不触发 T4；T1 单页足够，交通块默认打车 |
| 公交圈 | ≤ 12km | 公交 ~30min | 一般市内，T1 正常 |
| 分散 | > 12km | 跨区 | 提示是否按 §四 拆页 |

- 局限：直线距离 ≠ 实际步行/车程（有山/河/绕路会偏）；坐标缺失 >1 个 → 退回人工标记 `day.compact_area`（bool + 范围名）兜底。

### C · 新增字段 `day.on_trail_food`（唯一硬缺口·仅徒步日）
- **只围绕徒步/T3**，其他模板不用。
- 结构：`{ has_food: bool, supply_points: [{ name, type:'water'|'food'|'shop', note }] }`
- 来源：Stage 0 搜索（海外优先 YouTube 实拍）→ Stage 2 落库。
- 驱动：T3 是否含餐饮区块；无补给点 → 正餐丢到配套下一页。

## 九、点位计数规则 —— 按「目的」分，不按 type 一刀切

**核心：点位 = 为游玩/体验「专程去」的点，不是常规休息/前往/吃饭点。**

| 类别 | 是否算点位 |
|---|---|
| sight / sight_cultural / sight_nature / experience / experience_outdoor / experience_food / **shop** / venue / free | ✅ **恒为点位**（本身就是目的地；shop=专程逛街/买手店也算） |
| meal_* / hotel | ⚠️ **条件点位**：当 `intent ∈ {flagship, signature}` 或 `must_visit=true` → 算（专程去吃的名店、专程打卡的酒店）；否则常规吃饭/睡觉 → 不算 |
| transit / note | ❌ 永不计入 |

- 检测复用现有 `intent` + `must_visit`，无需大改；若 intent 标记不足以区分「专程 vs 常规」，加一个布尔 `is_destination`（为游玩体验而去）。
- 判定链：点位数 ≤3 → 宽松(T5/6/7)｜≥4 → 先看同质/集中(T4) 否则多点位(T1)。

## 十、Stage 4 = 完整输出结构（8 模板只是其中一段）

⚠️ Stage 4 不是「给每天选一个模板」那么简单——它要定**整份攻略的输出结构**，日页只是其中一段：

```
封面页
总览页（全程地图 + 每日一句 + 路线/住宿/预算）
─── 日页区（按行程形状，每天 1+ 页，从 T1–T8 选）───
  Day1 → [T?]…   Day2 → [T?]…   …
─── 图文内容页区（每类有数据才出页，可按拆页规则拆多页）───
  备选美食页（图文卡：图 + 名 + 介绍 + 价格/地点/提示）
  特产页    （图文卡：图 + 名 + 介绍 + 产地/价格/购买提示）
  （备选景点页 · 同款图文卡，可选）
附录页（纯文本参考：紧急联络 / 当地须知 / 行前清单 / 票据）
```

- **图文 vs 文本分离铁律**：带图带介绍的内容（特产 / 备选美食 / 备选景点）= **独立图文卡页**，绝不塞进附录；附录只放纯文本参考（紧急/须知/清单/票据）。
  - 依据：Stage 3 `supplement.template.html` 早已把特产/备选做成 `souvenir-card`/`card-thumb`+`card-intro` 图文卡（图 ~4:3 + 名称 + 一段介绍 + 产地/价格/提示）。Stage 4 必须沿用，不能降级成纯文字列表。
- 图文卡页结构：复用 T4 同款卡片网格（4:3/16:9 图 + 名 + 介绍 + meta 行）。
- 8 个模板 = 「日页区」的样式池；图文内容页 + 封面/总览/附录 = 文档级页面。
- **做 A（Stage 4 重写）时第一步：先定这份完整结构骨架，再把 8 模板嵌进日页区 + 写选择器 + 图文内容页。**

## 十一、Stage 4 重写架构（A 的蓝图）

### 输出文档结构
```
[1]   封面 cover                         ← 沿用现有 trip.template
[2]   总览 overview（全程地图+每日一句+路线/住宿/预算）← 沿用
[3..] 日页区：每个 day 经选择器 → 1+ 页（T1–T8）   ← 新（8 模板 + 选择器）
[末]  附录：须知 / 紧急联络 / 行前清单 / 伴手礼 / 票据  ← 沿用现有 4 modal + extras + checklist
```
- 复用现有：cover / overview / extras / checklist / 4 modal。
- 新建：T1–T8 日页模板 + `selectPagesForDay()` 选择器。

### 选择器算法（day → 页面序列，优先级判断）
```
selectPagesForDay(day):
  points    = timeline.filter(isPoint)              // §九 按目的计数
  crossSegs = timeline.filter(t.type=='transit' && t.intent=='intercity')
  hike      = timeline.find(t.type=='experience_outdoor' && isLongHike(t))

  1. points 为空 & 无实质活动            → [T8]
  2. crossSegs≥2 且 points≤1（纯赶路）   → [T2(全跨城)]
  3. crossSegs≥1：landing=末段跨城后的points
       landing≤2 → [T2(跨城+落地并入页底)]
       landing≥3 → [T2(跨城)] + selectInCity(landing)
  4. hike 存在：
       整天徒步 → [T3]
       半天徒步 → [T3] + selectInCity(其余points)
  5. 纯市内    → selectInCity(points)

selectInCity(points):
  points≤3 → [T5|T6|T7]（按数量）
  points≥4 → 同质(≥80%同category) 或 空间集中(步行圈≤1.5km) → splitPages(T4)
             否则 → splitPages(T1)

splitPages(points, tpl):  // §四 均衡拆页
  pages=ceil(N/5); 平均分(差值≤1); 切点靠时间段; 餐饮随覆盖时段进对应页
```

辅助判定：`isPoint`(§九) / `isHomogeneous`(§八A 映射后比例) / `isCompact`(§八B 坐标≤1.5km，缺坐标用 day.compact_area)。

### 待定判断点（建后即用，先确认）
- **isLongHike / 整天 vs 半天**：用徒步 `key_info.duration` —— **≥5h = 整天徒步(仅 T3)；3–5h = 半天(T3+其余页)；<3h** 当普通 experience_outdoor 点位，不单独成 T3 页。
  - 「占当天活动时长 >60% 升格为整天」**只在其它活动也标了 duration 时才生效**（否则分母只剩徒步会误判 100%）；缺时长时纯按上面小时阈值。
