# travel-itinerary · 旅游攻略 Skill for Claude Code

一个让 Claude Code 半自动生成旅行攻略的 Skill。核心做法是**围绕「地图 + 点位」做规划布局，再一键渲染出图**——你在地图上选点、排顺序、分天，Claude 帮你核实事实、补全信息、生成一份可打印可携带的攻略画册。

> 🎌 **历史样例**：[京都 5 天 4 人攻略](https://logic0512.github.io/kyoto-2026/)（v2.4.1 时代产物，由已退役的问卷流程生成；视觉与内容质量可参考，但**当前 v3.0 流程已改为下文的四界面地图管道**）

---

## 这版做了什么改变

旧版本（v2.x）靠两张问卷（Q1 决策 / Q2 行程锁定）让用户填表。**v3.0 彻底退役了问卷流程**，改成一条「发现 → 规划 → 补充 → 出图」的四界面地图管道。你不再面对表单，而是在地图上直接看着点位做决定，系统在背后承担研究、核实和排版。

## 工作流程：四界面地图管道

```
discovery          planner              supplement         trip-print-v2
发现选点      →    地图排点位 + 分天   →  勾选备选        →  8 模板渲染出图
（标必去/         （Leaflet 拖拽       （备选景点 /        （浏览器导出
  感兴趣）           排序 + 锁定）         美食 / 特产）        PDF 画册）
```

四个界面按数据流串联，每个都是双击即用的单文件 HTML，不依赖服务器：

1. **discovery 发现手册** — Claude 把研究出的景点 / 美食 / 体验候选注入，你三态标记「★必去 / ♡感兴趣 / —跳过」。
2. **planner 行程规划** — Leaflet 地图上拖拽排点位、分天、锁定动线；地图按当天点位自动聚焦。
3. **supplement 补充确认** — 从备选景点 / 美食 / 特产卡片库里勾选要补进攻略的。
4. **trip-print-v2 最终渲染** — Claude 合并前三个界面的产出，用 8 模板系统按「当天行程形状」自动选页，生成 HTML 画册，再由浏览器导出 PDF。

多城（≥2 城）和跨境行程走的是同一条管道，不另开分支：开局先定下「城市顺序 + 每城天数 + 城际交通段」骨架，planner 按天切换自然聚焦当天的城。

## 三条路径：按你的成熟度走不同界面

触发后，Claude 先做一次「路径分诊」，判断你处于哪种状态，再决定从哪个界面起步——不会把整套流程一股脑抛给你。

| 路径 | 适用 | 走哪几个界面 |
|---|---|---|
| **A · Full** | 没想法 / 不了解当地玩什么 | discovery → planner → supplement → trip-print-v2（全程） |
| **B · Constraint** | 有几个必去点和固定约束 | 跳过 discovery；planner 直接载入「你已有点 + Claude 补研究的点」→ supplement → trip-print-v2 |
| **C · Express** | 每天框架已基本定好 | 跳过 discovery + supplement；planner 仅核实排序 → trip-print-v2 |

判断维度包括目的地是否确定、你对目的地的**认知深度**（「知道去哪」≠「知道该怎么玩」）、日期、固定约束、必去点、行程成熟度。

---

## 三道开工前门：动手前先把前提核对清楚

v3.0 把质量门禁前移到「写任何行程之前」，避免硬伤一路绿灯产出错误攻略：

1. **事实自洽性预检** — 动手前先扫四类高频陷阱：目的地能不能串成一条动线（离群项？跨气候带？）、季节 × 出发地半球（南半球元旦是仲夏，不是冬天）、玩法 × 目的地可行性（腾冲不下雪就别安排「玩冰」）、具名实体真实位置（取景地 / 特定餐食是否真在计划城市）。命中疑点先核实、再回钩你确认，绝不静默修正或闷头照做。
2. **整趟可行性摊牌** — 把「地理 × 天数 / 预算 × 诉求 / 节奏 × 数量 / 关键岔路」四轴摆一桌，命中冲突就列出来 + 给两个取舍方案让你裁决，不闷头硬排、不偷偷砍诉求。
3. **出行资格准入** — 涉及自驾 / 跨境 / 特殊活动（潜水、入山徒步）时，查「人有没有资格用这种方式出行」：自驾查驾照承认 / 认证翻译件、跨境查签证有效期 / 护照 6 个月、特殊活动查执照许可，命中门槛提前告知 + 给补办路径或替代方案。

## 功能一览

- **四界面地图管道** — 发现选点 → 地图排点位 → 补充确认 → 8 模板渲染，全程围绕地图和点位，不再填表。
- **三路径分诊** — 按你的认知深度和计划成熟度，自动决定从哪个界面起步，不做无谓的步骤。
- **三道开工前门** — 事实预检 + 可行性摊牌 + 资格准入，把硬伤拦在动手之前。
- **贯穿约束清单** — 健康（高原反应）、安全（独行夜间）、预算档位、季节（梅雨备雨）、交通（包车 vs 班车）、同行结构（带娃 / 适老 / 多代）、主题朝圣（影视打卡 / 圣地巡礼）等横切关注点，由 Claude 全程把关，最终渲染进「出行须知」区。
- **多城城际骨架** — 开局固化城市顺序 + 每城天数 + 城际交通段，转场日自动出转场页；单个 planner 文件覆盖全程，不拆文件、不分图。
- **跨境出入境段** — 出发国 → 目的国的国际出入境段单独承载，海关申报、口岸到市区交通、中转行李直挂等提醒自动转写进攻略。
- **事实台账核查** — 每条关键事实（营业时间、定休日、票价、班次、坐标）记录来源 / 查询日期 / 置信度 / 对行程影响；用户提供的信息一律二次核实，发现不可行直接指出并给替代方案。
- **天气感知** — 出发前检测天气；planner 标室外点；成品页含天气栏 + 室内备选。
- **独行安全层** — 人数 = 1 时展开独行偏好 + 成品页夜间安全提醒。
- **HTML 优先，浏览器导出 PDF** — HTML/CSS 是唯一视觉排版源，PDF 必须由浏览器从 HTML 打印导出；禁止用代码直接绘制 PDF 或用色块假图冒充配图。
- **强制配图 + 资产台账** — PDF 中出现的非 transit 行程点必须有真实配图，并在 asset_ledger 追踪来源与状态。
- **任意目的地** — 国内 / 国际通用，链接策略自动切换（国内高德优先，国际 Google Maps / Tabelog），禁用大众点评 / 美团 / 小红书等强制登录链接。

## 安装

### 方式一：手动安装（推荐）

```bash
# 克隆此仓库
git clone https://github.com/logic0512/travel-itinerary.git

# 复制到 Claude Code skills 目录
cp -r travel-itinerary ~/.claude/skills/travel-itinerary
```

### 方式二：通过 Claude Code 插件市场

在 Claude Code 中运行：

```
/install-skill https://github.com/logic0512/travel-itinerary
```

## 使用方式

在 Claude Code 会话中说出你的旅行需求即可激活，例如：

> 「帮我做一个京都 5 天的攻略，5 月出发，4 个人」 —— 没具体想法，走 A 路径，Claude 先带你发现。

> 「我要去清迈 6 天，已经订了机票，想去大象保护营和周日夜市，其余帮我填」 —— 有部分计划，走 B 路径。

> 「这是我每天的安排，帮我核实一下排成能带着走的攻略」 —— 行程已定，走 C 路径。

Claude 会先做路径分诊和开工前预检，再按对应路径依次打开 discovery / planner / supplement 界面让你操作，最后渲染出 PDF 画册。

## 产物说明

| 产物 | 用途 |
|---|---|
| `discovery.html` | 发现手册，标记必去 / 感兴趣，导出选择列表 |
| `planner.html` | 地图行程规划，拖拽排点位、分天、锁定，导出行程结构 |
| `supplement.html` | 补充确认，勾选备选景点 / 美食 / 特产 |
| `trip-print-v2.html` | 最终视觉排版源（8 模板画册），浏览器打开即可验收 |
| `trip-guide.pdf` | 主成果，由浏览器从 HTML 导出，适合分享 / 携带 / 打印 |
| `fact_ledger.md` | 事实核查台账（来源 / 日期 / 置信度 / 影响） |
| `asset_ledger.md` | 图片资产台账（来源 / 状态 / 使用页） |

## 行程档位（Archetype）

| 档位 | 适用场景 |
|---|---|
| `short` | 1–3 天 · 单城周末游 |
| `medium-single` | 4–6 天 · 单城深度游 |
| `medium-multi` | 4–7 天 · 多城 / 跨城游 |
| `long` | 7 天以上 · 长途旅行 |

## 依赖

- **模型**：不指定，无硬绑定。
- **外部 skill**：无。主题生成用内置 `references/design-rules.md`。
- **所需工具**：`web_search`（信息核实与配图来源）+ `Read/Write/Edit/Bash`（构建与自检）。若环境支持浏览器截图 / PDF 渲染，最终出图会逐页截图验收。
